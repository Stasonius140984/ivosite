# 🔄 Рабочий процесс - Разработка и деплой ИЭО

## 📋 Обзор

Этот документ описывает полный цикл работы над проектом: от локальной разработки до публикации на production сервере Beget.

---

## 🏗️ Этапы разработки

### 1️⃣ Начало работы (один раз)

```bash
# Клонируйте проект (если используете Git)
git clone <repository-url>
cd ieo-rf

# Установите зависимости
npm install

# Запустите локальный сервер
npm run dev
```

**Результат:** Проект запущен на http://localhost:5173/

---

### 2️⃣ Локальная разработка

#### Ежедневный процесс:

```bash
# Запустите сервер разработки
npm run dev

# Внесите изменения в код
# Браузер автоматически обновится

# Когда закончили - остановите сервер
# Ctrl+C в терминале
```

#### Создание новой функции:

1. **Создайте ветку** (если используете Git):

   ```bash
   git checkout -b feature/новая-функция
   ```

2. **Внесите изменения:**
   - Отредактируйте существующие компоненты
   - Создайте новые компоненты в `components/`
   - Обновите стили в `styles/globals.css`

3. **Протестируйте локально:**
   - Проверьте на разных экранах (DevTools → Responsive)
   - Убедитесь, что нет ошибок в консоли (F12)
   - Проверьте навигацию между страницами

4. **Закоммитьте** (если используете Git):
   ```bash
   git add .
   git commit -m "Добавлена новая функция"
   ```

---

### 3️⃣ Тестирование перед деплоем

---

### 4️⃣ Деплой на Beget

#### Способ 1: Через FTP-клиент (FileZilla)

```bash
# 1. Соберите проект
npm run build

# 2. Откройте FileZilla
# 3. Подключитесь к Beget:
#    - Хост: ft*.beget.tech
#    - Порт: 21 (FTP) или 22 (SFTP)
#    - Логин: ваш логин
#    - Пароль: ваш пароль

# 4. Перейдите в public_html/

# 5. Удалите старые файлы (кроме .htaccess)

# 6. Загрузите всё из dist/:
#    - index.html
#    - папку assets/
#    - favicon.svg
#    - все остальные файлы

# 7. Убедитесь, что .htaccess на месте
```

#### Способ 2: Через файловый менеджер Beget

```bash
# 1. Соберите проект
npm run build

# 2. Войдите в панель Beget
# 3. Откройте "Файловый менеджер"
# 4. Перейдите в public_html/
# 5. Нажмите "Загрузить файлы"
# 6. Выберите всё из папки dist/
# 7. Дождитесь завершения загрузки
```

### 5️⃣ Проверка на production

```bash
# Откройте сайт в браузере
https://иэо.рф

# Проверьте:
# - Главная страница
# - Все категории (через меню "Категории")
# - "Для бизнеса"
# - "О нас"
# - Адаптивность (откройте DevTools)
# - SSL (должен быть замок в адресной строке)
```

**Очистите кеш:**

- Windows/Linux: `Ctrl + Shift + Delete`
- macOS: `Cmd + Shift + Delete`

## 📊 Git workflow (опционально)

### Базовый процесс

```bash
# Обновить локальную версию
git pull origin main

# Создать ветку для новой функции
git checkout -b feature/новая-функция

# Внести изменения и закоммитить
git add .
git commit -m "Описание изменений"

# Отправить на сервер
git push origin feature/новая-функция

# Создать Pull Request на GitHub/GitLab

# После одобрения - слить в main
git checkout main
git merge feature/новая-функция

# Деплой
npm run build
# Загрузить на Beget
```

## 🚨 Экстренные ситуации

### Сайт не работает после обновления

```bash
# 1. Откройте консоль браузера (F12)
# 2. Посмотрите ошибки
# 3. Проверьте, что все файлы загружены

# Быстрое решение:
# - Загрузите предыдущую версию из dist/
# - Или откатите Git: git checkout <previous-commit>
# - Пересоберите: npm run build
# - Загрузите на сервер
```

### Пропал .htaccess

```bash
# 1. Загрузите .htaccess из корня проекта
# 2. Убедитесь, что права доступа: 644
# 3. Очистите кеш браузера
```

### Стили не применяются

```bash
# 1. Очистите кеш браузера (Ctrl+Shift+Delete)
# 2. Проверьте, что файлы CSS загружены в assets/
# 3. Проверьте пути в index.html
# 4. Пересоберите проект:
npm run build
# Загрузите заново
```

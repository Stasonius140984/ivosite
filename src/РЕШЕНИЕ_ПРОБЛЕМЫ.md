# ✅ РЕШЕНИЕ ПРОБЛЕМЫ С ХЕШАМИ ФАЙЛОВ

## 🔴 Проблема

Вы столкнулись с ситуацией, когда:
- Браузер запрашивает файлы: `index-Blu3TnO_.js` и `index-CCGmDF7H.css`
- На сервере лежат файлы с другими хешами (старая сборка)
- Результат: **ошибки 404**, сайт не работает

## 💡 Суть проблемы

При сборке проекта Vite генерирует уникальные хеши для файлов:
```
build/assets/index-[ХЕШ].js
build/assets/index-[ХЕШ].css
```

Эти хеши **меняются при каждой сборке**. Файл `index.html` содержит ссылки на эти файлы с текущими хешами.

**Если на сервере старая сборка**, то:
1. Браузер загружает `index.html` с **новыми** хешами
2. Но файлы на сервере имеют **старые** хеши
3. → **404 Not Found**

## ✅ Решение

Нужно **пересобрать проект** и **загрузить свежую сборку** на сервер.

---

## 🚀 Быстрые решения (выберите одно)

### 📋 Вариант 1: Скрипты (самое простое)

#### Windows:
```cmd
build-local.bat
```
Затем следуйте инструкциям в `ИНСТРУКЦИЯ_WINSCP.md`

#### Linux/Mac:
```bash
chmod +x build-local.sh deploy-build.sh
./build-local.sh
./deploy-build.sh
```

**Время:** 3-5 минут

---

### ⚡ Вариант 2: Одна команда

Если у вас Linux/Mac и настроен SSH:

```bash
npm run build && scp -r build/* root@217.114.15.80:/var/www/ivo/build/ && ssh root@217.114.15.80 "chown -R www-data:www-data /var/www/ivo/build/ && chmod -R 755 /var/www/ivo/build/ && systemctl restart nginx"
```

**Время:** 2-3 минуты

---

### 🔧 Вариант 3: Сборка на сервере

Если исходники уже на сервере (файлы `*.tsx`, `package.json`, и т.д.):

```bash
ssh root@217.114.15.80 "cd /var/www/ivo && rm -rf build && npm install && npm run build && chown -R www-data:www-data build/ && chmod -R 755 build/ && systemctl restart nginx"
```

**Время:** 2-3 минуты

### 📝 Вариант 4: Вручную (пошагово)

Если хотите понять процесс:

```bash

```

Результат: создастся папка `build/` с файлами.

#### Шаг 2: Загрузка на сервер

**Через SCP (Linux/Mac):**
```bash

```

**Через WinSCP/FileZilla (Windows):**
1. Подключиться к серверу `217.114.15.80`
2. Удалить старую папку `/var/www/ivo/build/`
3. Загрузить новую папку `build/`

#### Шаг 3: Права доступа (на сервере)
```bash
ssh root@217.114.15.80
chown -R www-data:www-data /var/www/ivo/build/
chmod -R 755 /var/www/ivo/build/
systemctl restart nginx
```

1. Откройте: http://иэо.рф

## 🆘 Если не помогло

### Проблема: Всё равно 404

**Диагностика:**
```bash
# Проверьте, какие файлы на сервере
ssh root@217.114.15.80 "ls -la /var/www/ivo/build/assets/"

# Проверьте, какие файлы запрашивает index.html
ssh root@217.114.15.80 "cat /var/www/ivo/build/index.html | grep assets"
```

**Решение:** Полная пересборка
```bash
ssh root@217.114.15.80 "cd /var/www/ivo && rm -rf build node_modules && npm install && npm run build && chown -R www-data:www-data build/ && chmod -R 755 build/ && systemctl restart nginx"
```

### Проблема: Стили не применяются

**Причина:** Кеш браузера

**Решение:**
1. Ctrl+Shift+R (жёсткая перезагрузка)
2. Очистите кеш полностью в настройках браузера
3. Откройте в режиме инкогнито

### Проблема: npm run build падает

**Решение:**
```bash
rm -rf node_modules package-lock.json
npm install
npm run build
```

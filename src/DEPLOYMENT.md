# 🚀 Инструкция по размещению на Beget

## Шаг 1: Подготовка проекта

### 1.1 Установка зависимостей (если ещё не установлены)
```bash
npm install
```

### 1.2 Сборка проекта
```bash
npm run build
```

После выполнения команды будет создана папка `dist` со всеми файлами сайта.

## Шаг 2: Регистрация домена иэо.рф

### 2.1 Покупка домена
1. Зайдите на сайт регистратора доменов:
   - **nic.ru** (официальный регистратор .рф доменов)
   - **REG.RU**
   - Можно купить сразу на Beget

2. Найдите домен **иэо.рф** (или **xn--h1aaa.xn--p1ai** в Punycode)
3. Оформите покупку

### 2.2 Настройка DNS
В панели управления доменом укажите NS-серверы Beget:
```
ns1.beget.com
ns2.beget.com
ns1.beget.ru
ns2.beget.ru
```

## Шаг 3: Настройка хостинга Beget

### 3.1 Регистрация на Beget
1. Зайдите на [beget.com](https://beget.com)
2. Выберите тариф (для вашего сайта подойдёт "Начальный" или "Оптимальный")
3. Зарегистрируйтесь и оплатите

### 3.2 Добавление домена в панели Beget
1. Войдите в панель управления Beget
2. Перейдите в раздел **"Домены и поддомены"**
3. Нажмите **"Добавить домен"**
4. Введите **иэо.рф** (или xn--h1aaa.xn--p1ai)
5. Выберите папку для сайта (например, `/иэо.рф/public_html`)

### 3.3 Включение HTTPS
1. В панели Beget перейдите в **"SSL-сертификаты"**
2. Выберите ваш домен **иэо.рф**
3. Нажмите **"Заказать бесплатный Let's Encrypt"**
4. Дождитесь выпуска сертификата (обычно 5-10 минут)

## Шаг 4: Загрузка файлов на хостинг

### Вариант A: Через FTP (рекомендуется)

#### 4.1 Получение данных FTP
В панели Beget найдите:
- **FTP-сервер**: обычно `имя_аккаунта.beget.tech`
- **Логин**: ваш логин Beget
- **Пароль**: ваш пароль Beget

#### 4.2 Подключение через FileZilla
1. Скачайте [FileZilla](https://filezilla-project.org/)
2. Подключитесь:
   - Хост: `ftp://ваш_логин.beget.tech`
   - Логин: ваш логин
   - Пароль: ваш пароль
   - Порт: 21

#### 4.3 Загрузка файлов
1. В FileZilla откройте папку вашего домена:
   - Обычно: `/иэо.рф/public_html/` или `/xn--h1aaa.xn--p1ai/public_html/`
2. На локальном компьютере откройте папку `dist` вашего проекта
3. Выделите **все файлы и папки** внутри `dist`
4. Перетащите их в папку `public_html` на сервере
5. Дождитесь завершения загрузки

### Вариант B: Через файловый менеджер Beget

1. В панели Beget откройте **"Файловый менеджер"**
2. Перейдите в папку `/иэо.рф/public_html/`
3. Удалите все существующие файлы (если есть)
4. Нажмите **"Загрузить"**
5. Выберите все файлы из папки `dist` на вашем компьютере
6. Дождитесь загрузки

## Шаг 5: Проверка файла .htaccess

Файл `.htaccess` уже создан в корне проекта и будет в папке `dist` после сборки.

Проверьте, что файл `.htaccess` присутствует в папке `public_html` на сервере:
- Если его нет, создайте его вручную через файловый менеджер Beget
- Скопируйте содержимое из файла `.htaccess` в корне проекта

## Шаг 6: Проверка работы сайта

1. Откройте браузер
2. Перейдите на **https://иэо.рф** (или https://xn--h1aaa.xn--p1ai)
3. Проверьте все страницы:
   - Главная
   - Граждане
   - Сотрудник
   - Охрана
   - ГБР
   - Росгвардия

## Шаг 7: Настройка редиректа с www

В файловом менеджере Beget добавьте в начало файла `.htaccess`:

```apache
# Редирект с www на основной домен
RewriteCond %{HTTP_HOST} ^www\.иэо\.рф$ [NC]
RewriteRule ^(.*)$ https://иэо.рф/$1 [R=301,L]
```

## Возможные проблемы и решения

### Проблема: Сайт не открывается
**Решение**: 
- Проверьте, что DNS-записи обновились (может занять до 24 часов)
- Проверьте командой: `ping иэо.рф`

### Проблема: Страницы не работают при прямом переходе
**Решение**: 
- Убедитесь, что файл `.htaccess` загружен и находится в корне `public_html`
- Проверьте права на файл (должны быть 644)

### Проблема: Стили не загружаются
**Решение**: 
- Очистите кэш браузера (Ctrl+F5)
- Проверьте, что все файлы из папки `dist` загружены

### Проблема: HTTPS не работает
**Решение**: 
- Подождите 10-15 минут после заказа сертификата
- Если не помогло, переоформите сертификат в панели Beget

## Обновление сайта в будущем

Когда нужно обновить сайт:

1. Внесите изменения в код
2. Выполните `npm run build`
3. Удалите старые файлы из `public_html` на Beget
4. Загрузите новые файлы из папки `dist`

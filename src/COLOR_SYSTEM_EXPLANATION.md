## 🏗️ Архитектура системы цветов

### Уровень 1: CSS переменные (:root)
```css
:root {
  --tricolor-blue: #0039A6;
  --primary: #0039A6;
  /* ... и т.д. */
}
```

### Уровень 2: Tailwind токены (@theme inline)
```css
@theme inline {
  --color-tricolor-blue: var(--tricolor-blue);
  --color-primary: var(--primary);
  /* ... и т.д. */
}
```

### Уровень 3: Tailwind классы (используются в компонентах)
```tsx
// Стандартные Tailwind классы (работают сейчас)
<div className="bg-blue-900 text-white">

// Кастомные классы через переменные (можно использовать)
<div className="bg-primary text-primary-foreground">
<div className="bg-tricolor-blue text-white">
```

---

## 🔄 Почему сейчас работают цвета

Ваши компоненты используют **стандартные Tailwind классы**:
- `bg-blue-900` → #1e3a8a (темно-синий Tailwind)
- `bg-blue-700` → #1d4ed8 (синий Tailwind)
- `bg-red-700` → #b91c1c (красный Tailwind)
- `text-white` → #ffffff

Эти классы работают благодаря `@import "tailwindcss";`

## 🎯 Два подхода использования

### Подход 1: Стандартные Tailwind классы (текущий)

**Преимущества:**
- ✅ Работает из коробки
- ✅ Не нужно дополнительно настраивать
- ✅ Понятные названия классов

**Недостатки:**
- ❌ Цвета не совпадают с официальным триколором (немного отличаются)
- ❌ Не используются ваши кастомные переменные

**Используется в компонентах:**
```tsx
<header className="bg-gradient-to-r from-blue-900 via-blue-800 to-blue-900">
<div className="bg-blue-700 text-white">
<Button className="bg-red-700">
```

---

### Подход 2: Кастомные переменные (рекомендуется для бренда)

**Преимущества:**
- ✅ Точные цвета российского триколора
- ✅ Единая система цветов
- ✅ Легко менять цвета глобально
- ✅ Поддержка темной темы

**Недостатки:**
- ⚠️ Нужно обновить все компоненты
- ⚠️ Требует больше настроек

**Как использовать:**
```tsx
// Через Tailwind классы (автоматически из переменных)
<header className="bg-primary text-primary-foreground">
<div className="bg-tricolor-blue text-white">
<Button className="bg-destructive text-destructive-foreground">

// Или напрямую через CSS
<div style={{ backgroundColor: 'var(--tricolor-blue)' }}>
```

## 📝 Техническая информация

### Файлы конфигурации:

| Файл | Назначение |
|------|------------|
| `styles/globals.css` | CSS переменные + импорт Tailwind |
| `vite.config.ts` | Конфигурация сборки |
| `postcss.config.js` | PostCSS настройки |
| `package.json` | Зависимости (Tailwind CSS v4) |

### Как работает компиляция:

1. Vite читает `main.tsx`
2. Импортирует `styles/globals.css`
3. PostCSS обрабатывает `@import "tailwindcss"`
4. Tailwind генерирует CSS из классов в компонентах
5. Применяет переменные из `@theme inline`
6. Минифицирует и сохраняет в `dist/assets/`

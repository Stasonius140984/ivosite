## ⚡ Быстрое решение (выберите свой вариант)

### 🪟 Вариант A: У вас Windows?

1. **Откройте PowerShell** в папке проекта
2. **Выполните:**
   ```cmd
   build-local.bat
   ```
3. **Дождитесь** окончания сборки (1-3 минуты)
4. **Откройте файл:** `ИНСТРУКЦИЯ_WINSCP.md`
5. **Следуйте инструкциям** по загрузке через WinSCP

**Время:** 5-10 минут

---

### 🐧 Вариант B: У вас Linux/Mac?

1. **Откройте терминал** в папке проекта
2. **Выполните:**
   ```bash
   chmod +x build-local.sh deploy-build.sh
   ./build-local.sh
   ./deploy-build.sh
   ```
3. **Всё!** Сайт обновлён

**Время:** 3-5 минут

---

### ⚙️ Вариант C: Хотите одну команду?

**Если у вас Linux/Mac и настроен SSH:**

```bash
npm run build && scp -r build/* root@217.114.15.80:/var/www/ivo/build/ && ssh root@217.114.15.80 "chown -R www-data:www-data /var/www/ivo/build/ && chmod -R 755 /var/www/ivo/build/ && systemctl restart nginx"
```

Скопируйте, вставьте, нажмите Enter. Готово!

**Время:** 2-3 минуты

---

### 🔧 Вариант D: Исходники уже на сервере?

Если все файлы проекта уже загружены на сервер, выполните:

```bash
ssh root@217.114.15.80 "cd /var/www/ivo && rm -rf build && npm install && npm run build && chown -R www-data:www-data build/ && chmod -R 755 build/ && systemctl restart nginx"
```

**Время:** 2-3 минуты

## 🆘 Что-то не работает?

### Проблема: build-local.bat не запускается

**Решение:** Установите Node.js с https://nodejs.org/

### Проблема: Ошибка при сборке

**Решение:** 
```bash
rm -rf node_modules package-lock.json
npm install
npm run build
```

### Проблема: Не могу подключиться к серверу

**Решение:** Проверьте пароль и доступ к серверу 217.114.15.80

### Проблема: После деплоя стили не применяются

**Решение:** Очистите кеш браузера полностью:
- Chrome: Настройки → Конфиденциальность → Очистить данные
- Firefox: Настройки → Приватность → Очистить данные
- Или откройте в режиме инкогнито

### Проблема: В консоли браузера ошибки 404

**Решение:** Файлы не загружены или загружены неправильно. Повторите процесс.

## 🎯 Суть проблемы

Vite при сборке создаёт файлы с уникальными хешами:
- `index-Blu3TnO_.js`
- `index-CCGmDF7H.css`

Эти хеши меняются при каждой сборке. Файл `index.html` содержит ссылки на эти файлы.

**Если на сервере старая сборка**, браузер запрашивает новые хеши, а сервер их не находает → **ошибка 404**.

**Решение:** Пересобрать проект и загрузить новую сборку на сервер.

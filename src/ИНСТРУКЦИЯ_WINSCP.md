# 🪟 Инструкция по деплою через WinSCP (Windows)

## 📌 Краткое описание

Эта инструкция для пользователей **Windows**, которые хотят собрать проект локально и загрузить на сервер через **WinSCP** или **FileZilla**.

---

## 🎯 Пошаговая инструкция

### Шаг 1: Подготовка (один раз)

#### 1.1. Установите необходимые программы

- **Node.js** - https://nodejs.org/ (выберите LTS версию)
- **WinSCP** - https://winscp.net/eng/download.php
  - Или **FileZilla** - https://filezilla-project.org/

#### 1.2. Проверьте установку

Откройте **PowerShell** или **Command Prompt** и введите:

```cmd
node -v
npm -v
```

Вы должны увидеть версии Node.js и npm. Если нет - перезагрузите компьютер.

---

### Шаг 2: Локальная сборка проекта

#### 2.1. Откройте папку проекта

В **PowerShell** или **Command Prompt** перейдите в папку с проектом:

```cmd
cd C:\путь\к\вашему\проекту
```

Например:
```cmd
cd C:\Users\YourName\Projects\ieo-rf
```

#### 2.2. Запустите скрипт сборки

**Способ 1: Через готовый скрипт**

```cmd
build-local.bat
```

**Способ 2: Вручную**

```cmd
npm install
npm run build
```

#### 2.3. Дождитесь завершения

Процесс займёт 1-3 минуты. В конце вы увидите:

```
✅ Проект успешно собран!
📁 Содержимое папки build/:
...
🎉 Готово к загрузке!
```

---

### Шаг 3: Подключение к серверу через WinSCP

#### 3.1. Откройте WinSCP

Запустите программу **WinSCP**.

#### 3.2. Создайте новое подключение

Нажмите **"Новое подключение"** и введите:

| Параметр          | Значение           |
|-------------------|--------------------|
| **Протокол**      | SFTP               |
| **Имя хоста**     | 217.114.15.80      |
| **Порт**          | 22                 |
| **Имя пользователя** | root            |
| **Пароль**        | *ваш пароль*       |

Нажмите **"Сохранить"**, затем **"Войти"**.

#### 3.3. Подтвердите подключение

При первом подключении WinSCP спросит о доверии к серверу. Нажмите **"Да"**.

---

### Шаг 4: Загрузка файлов на сервер

#### 4.1. Откройте две панели

После подключения вы увидите две панели:
- **Слева** - ваш локальный компьютер
- **Справа** - удалённый сервер

#### 4.2. Перейдите в нужные папки

**Слева (локально):**
- Перейдите в папку вашего проекта
- Откройте папку `build`

**Справа (на сервере):**
- Перейдите в `/var/www/ivo/`

#### 4.3. Удалите старую папку build (ВАЖНО!)

**На сервере** (правая панель):
1. Найдите папку `build`
2. Кликните правой кнопкой мыши → **"Удалить"**
3. Подтвердите удаление

#### 4.4. Загрузите новую папку build

**На локальном компьютере** (левая панель):
1. Выделите **содержимое** папки `build` (index.html, favicon.svg, папку assets)
2. Кликните правой кнопкой → **"Загрузить"**
3. В диалоге выберите `/var/www/ivo/build/` как целевую папку
4. Нажмите **"ОК"**

Или проще:
1. На сервере создайте новую пустую папку `build`
2. Перетащите **все файлы** из локальной папки `build` в серверную папку `build`

#### 4.5. Дождитесь завершения загрузки

Загрузка займёт 1-2 минуты в зависимости от скорости интернета.

---

### Шаг 5: Установка прав доступа (через PuTTY)

#### 5.1. Откройте PuTTY

Если у вас нет PuTTY, скачайте его: https://www.putty.org/

#### 5.2. Подключитесь к серверу

В PuTTY введите:
- **Host Name:** 217.114.15.80
- **Port:** 22
- Нажмите **"Open"**

Введите:
- **login as:** root
- **password:** *ваш пароль*

#### 5.3. Выполните команды

Скопируйте и вставьте (правой кнопкой мыши в PuTTY вставляет):

```bash
chown -R www-data:www-data /var/www/ivo/build/
```

Нажмите **Enter**.

Затем:

```bash
chmod -R 755 /var/www/ivo/build/
```

Нажмите **Enter**.

Затем:

```bash
systemctl restart nginx
```

Нажмите **Enter**.

#### 5.4. Проверьте результат

Вы должны увидеть что-то вроде:

```
✅ Готово!
```

Если есть ошибки, напишите их текст для диагностики.

---

### Шаг 6: Проверка сайта

#### 6.1. Откройте сайт

В браузере откройте:

```
http://иэо.рф
```

#### 6.2. Очистите кеш

Нажмите **Ctrl+Shift+R** (или **Ctrl+F5**) для жёсткой перезагрузки.

#### 6.3. Проверьте работоспособность

Сайт должен:
- ✅ Открыться без ошибок
- ✅ Показать синий header (#0039A6)
- ✅ Показать триколор в hero секциях
- ✅ Все страницы работают (граждане, сотрудники, и т.д.)
- ✅ Навигация работает

#### 6.4. Проверьте консоль (опционально)

Нажмите **F12** → вкладка **Network** → перезагрузите страницу.

Убедитесь, что все файлы загружаются с кодом **200 OK**, а не **404**.

## 🆘 Решение проблем

### Проблема: WinSCP не подключается

**Возможные причины:**
- Неправильный пароль
- Порт 22 заблокирован фаерволом
- Сервер недоступен

**Решение:**
1. Проверьте пароль
2. Проверьте, что используете протокол **SFTP**, а не FTP
3. Попробуйте подключиться через другую сеть (например, мобильный интернет)

### Проблема: Не удаётся удалить старую папку build

**Причина:** Нет прав доступа

**Решение:**
1. Подключитесь через PuTTY
2. Выполните:
   ```bash
   rm -rf /var/www/ivo/build/
   mkdir /var/www/ivo/build/
   chmod 755 /var/www/ivo/build/
   ```
3. Теперь загрузите файлы через WinSCP

### Проблема: После загрузки сайт не работает (404)

**Причина:** Не установлены права доступа или nginx не перезапущен

**Решение:**
Выполните в PuTTY:
```bash
chown -R www-data:www-data /var/www/ivo/build/
chmod -R 755 /var/www/ivo/build/
systemctl restart nginx
```

Затем очистите кеш браузера: **Ctrl+Shift+R**

### Проблема: Стили не применяются

**Причина:** Кеш браузера

**Решение:**
1. **Ctrl+Shift+R** (жёсткая перезагрузка)
2. Или откройте в режиме инкогнито
3. Или очистите кеш в настройках браузера:
   - Chrome: Настройки → Конфиденциальность → Очистить данные просмотра
   - Firefox: Настройки → Приватность и защита → Очистить данные
   - Edge: Настройки → Конфиденциальность → Выберите, что нужно очистить

# 🚀 Быстрый старт - Деплой на Beget

## Пошаговая инструкция (5 минут)

### 1️⃣ Соберите проект

```bash
# Установите зависимости
npm install

# Соберите production версию
npm run build
```

После выполнения появится папка `dist/` с готовым сайтом.

---

### 2️⃣ Подготовьте Beget

1. Войдите в панель управления: https://beget.com
2. Перейдите в **"Сайты и домены"**
3. Добавьте домен: **xn--d1a4b.xn--p1ai** (это Punycode для иэо.рф)
4. Привяжите к папке `public_html`

---

### 3️⃣ Загрузите файлы

**Через Файловый менеджер Beget:**
1. Откройте "Файловый менеджер"
2. Перейдите в `домен/public_html/`
3. Загрузите **всё содержимое** папки `dist/`:
   - `index.html`
   - папку `assets/`
   - все остальные файлы

**ИЛИ через FTP (FileZilla):**
1. Подключитесь к FTP Beget
2. Перетащите всё из `dist/` в `public_html/`

---

### 4️⃣ Загрузите .htaccess

Загрузите файл `.htaccess` из корня проекта в папку `public_html/`

Этот файл обеспечивает:
- Работу роутинга React
- Редирект на HTTPS
- Кеширование
- Безопасность

---

### 5️⃣ Настройте SSL

1. В панели Beget: **"SSL-сертификаты"**
2. Выберите домен **иэо.рф**
3. Нажмите **"Установить бесплатный SSL"**
4. Подождите 5-15 минут

---

### ✅ Проверка

Откройте в браузере:
- https://иэо.рф
- https://xn--d1a4b.xn--p1ai

Должна загрузиться главная страница с триколором! 🇷🇺

---

## 🔄 Обновление сайта

Когда нужно обновить сайт:

```bash
# 1. Соберите новую версию
npm run build

# 2. Загрузите новые файлы из dist/ на сервер
# 3. Очистите кеш браузера (Ctrl+F5)
```

---

## 📁 Структура на сервере

```
public_html/
├── index.html          ← Главный HTML файл
├── .htaccess           ← Конфигурация сервера
├── favicon.svg         ← Иконка сайта
└── assets/             ← JS, CSS, изображения
    ├── index-abc123.js
    ├── index-def456.css
    └── ...
```

## ⚠️ Важно

- **НЕ загружайте** саму папку `dist/` - только её содержимое!
- **ОБЯЗАТЕЛЬНО** загрузите `.htaccess` - без него навигация не работает
- Домен в Punycode: **иэо.рф** = **xn--d1a4b.xn--p1ai**

## 🆘 Проблемы?

### Белый экран
- Проверьте консоль браузера (F12)
- Убедитесь, что файлы загружены в `public_html/`, а не в подпапку

### 404 при переходах
- Проверьте наличие `.htaccess`
- Права доступа к `.htaccess` должны быть 644

### SSL не работает
- Подождите 15-20 минут после активации
- Проверьте настройки в разделе "SSL-сертификаты"
